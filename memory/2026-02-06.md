## 🔧 Fast Note Sync 笔记同步问题

### 问题现象
- cron 脚本在 8:30 创建了工作笔记
- 笔记写入文件系统但 Fast Note Sync 无法同步
- Mac Obsidian 看不到自动生成的笔记

### 问题根源
**Mac 和服务器的 Vault 名称不匹配**

| 位置 | Vault Name | 状态 |
|------|------------|------|
| **服务器** | `Obsidian Vault` | ✅ 工作笔记存在 |
| **Mac** | `obsidian` | ❌ 名称不匹配 |

### 解决方案
1. **修改 cron 脚本** - 使用 API 创建笔记而非直接写文件
2. **Mac 需要修改配置** - Vault Name 改为 `Obsidian Vault`

### 修复后的 cron 脚本
```bash
# 正确的 API 调用方式
POST /api/note
  vault=Obsidian Vault  ← 仓库名称（字符串）
  path=n_910.md
  content=笔记内容
```

### Fast Note Sync API 关键接口
- `POST /api/user/login` - 登录获取 token
- `GET /api/vault` - 获取 vault 列表
- `POST /api/note` - 创建/更新笔记（需要 vault 参数）
