## Power BI 自动邮件提醒方案

### 需求背景
- Power BI 报表中有销售数据（销售名、保险金额、账款金额）
- 当账款达到保险金额 80% 时，自动发邮件给对应销售
- 销售**不需要** Power BI License

### 研究的方案

| 方案 | 安全性 | 复杂度 | 推荐度 |
|------|--------|--------|--------|
| **Power BI 数据警报** | ✅ 最高（数据不离开微软云） | ⭐ 简单 | ✅ **推荐** |
| **Power BI 订阅** | ✅ 高 | ⭐ 简单 | ⭐⭐ |
| **Power Automate** | ⚠️ 需配置 | ⭐⭐⭐ | ⭐⭐ |
| **n8n** | ❌ 数据需传到服务器 | ⭐⭐ | ❌ 不推荐 |
| **SharePoint + Power Automate** | ✅ 企业内 | ⭐⭐⭐ | ⭐⭐ |

### 最终推荐方案：Power BI 数据警报

**优点：**
- ✅ 数据永远在 Power BI 云中，不离开微软管控
- ✅ 不需要额外服务器
- ✅ 销售无需 License，邮件直接发送到邮箱
- ✅ 企业 IT 可管控
- ✅ 免费
- ✅ 设置简单

**实现步骤：**
1. 在 Power BI 报表中固定达成率可视化到仪表板
2. 仪表板图表右上角 "..." → "管理警报" → "添加警报规则"
3. 设置阈值 80%，勾选"同时发送邮件"
4. 分享仪表板给销售（只读权限）

**局限：**
- 警报是全局的，不能按行精细控制不同销售发不同邮件

**如需精细控制：**
- 使用企业内 SharePoint + Power Automate（需要 IT 协助配置）

### n8n 检查结果
- n8n Docker 容器已在服务器运行（端口 5678）
- 但由于需要连接数据库，不符合数据安全要求（数据需传到服务器）
- 决定不使用 n8n
